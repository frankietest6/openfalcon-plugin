OpenFalcon Plugin Configuration

Manage your connection, playlist, and listener · v

Server Connection

Server URL
Where your self-hosted OpenFalcon server is reachable. e.g. http://192.168.1.230:3100 or https://lights.yourdomain.com. No trailing slash.
Show Token
Copy this from your OpenFalcon admin dashboard ("Show Token (for OpenFalcon Plugin)" section).

Playlist Setup

Remote Playlist
The FPP playlist that contains the pool of sequences your viewers can vote for or request.
Sync Sequences with OpenFalcon
Reads the selected playlist and pushes its sequences to OpenFalcon. Run this whenever you add or remove sequences from the playlist.

Behavior

Interrupt Schedule
Determines if a request or vote will interrupt the normal schedule. Required if your remote playlist is a separate "pool" of viewer-controllable songs (rather than the playlist that's currently playing).
Request/Vote Fetch Time
Number of seconds before the end of the current sequence to fetch the next vote winner / jukebox request. Default: 3. Range: 1–5.
Additional Wait Time
Extra seconds to wait after queuing a sequence before checking for the next one. Helpful if you're seeing requests get skipped or duplicated. Default: 0.
FPP Status Check Time
How often the plugin polls FPP's status API. Lower = more responsive, higher CPU. Default: 1 (second).

Listener Control

Restart Listener
Reloads the plugin config and restarts the polling loop. Run this after changing config values via "Plugin Config" below.
Stop Listener
WARNING: This stops the plugin entirely. No requests/votes will be fetched and no playback state will be reported to OpenFalcon.

Diagnostics

Test Connectivity
Pings the OpenFalcon server using the Show Token to verify it's reachable and the token is correct.
Listener Log (tail)
Latest lines from /home/fpp/media/logs/openfalcon-listener.log. Auto-refreshes every 5 seconds while shown.
